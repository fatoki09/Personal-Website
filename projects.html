A closer look at some of my independent and school projects.

Find Doc

Created on December 2012 using JQuery Mobile, PHP, and MySQL.


Find Doc is a web app created for Stanford's CS 147 introduction to HCI class. The app helps users find the best doctor for them based on their symptoms, location, and overall ratings. This web application was built by Vivek Jain, Cory Weinstein, and myself using a JQuery mobile framework along with some PHP and MySQL. Over this 10-week course, I truly developed a design mindset. I learned about needfinding, rapid prototyping, and user-testing early on. I also had the opportunity to dive into a mobile web development framework and pick up some back-end skills along the way. I was even able to present the app to a panel of judges including Instagram co-founder Mike Krieger at the end of the year. Talk about an awesome experience! You can find it here.

Heap Allocator

Created on December 2012 using C.


My heap allocator manages the heap and implements the dynamic allocation functions malloc, realloc, and free. The allocator grabs a large memory segment using the low-level OS memory routines and parcels out this segment to service requests for allocation, reallocation, and deallocation. In addition, it manages the heap segment, tracks the available/in-use space, and updates the heap data structures in response to requests. The allocator runs at approximately 80% utilization and 50% throughput compared to the standard libc allocator. You can email me here if you'd like to see the code.

Holly Jachowski and I created this allocator as the final assignment for our CS 107 class. The assignment probably took nearly 40 hours and was truly a learning experience. There are several approaches to designing an allocator like this one and we had to carefully assess all of the trade-offs and advantages of switching between different approaches. We ultimately stuck with a buckets and doubly-linked list approach where we separated blocks of memory by size and stored those blocks in a doubly-linked list. We encountered several segfaults and other bugs and had a tough time debugging due to the inability to use gdb and valgrind. However, through perseverance and SEVERAL sleepless nights, we finally finished a project we can truly be proud of.
